PROBLEM

Navigation

Users experience difficulty navigating and accessing information due to the sloppy menu which leads to frustration.

Content

GSS website offers rich content; however it is not well structured, prioritized and connected to other relevant content that curb the easy access to information.

Irresponsiveness

The website is not responsive which makes it hard for user to use through mobile or tab.

Dynamic Interaction

The website lacks interactivity and dynamic interaction where it’s necessary for convenient user interaction.

Engagement

Users get brutal experience while browsing the website for not reaching out to their desired information.

Dynamic Front-End Content

No system to automatically add content to front-end from backend input by generating dynamic design patterns.

BACKGROUND

BACKGROUND

We were provided with the Clients requirements and expectations, overall user needs and experiences. Additionally I have interviewed three regular users of the website to understand what pain point they are facing while interacting with the website.

Usability Testing

We have conducted usability testing of the website at this stage to better understand the current problems of the website in order to mitigate those in the updated design.

By analyzing the client and user needs, and usability testing we came up with the solutions to address the current website problems.

PROJECT GOALS

The ultimate goal of this project is to create a clean concise and responsive website in order to provide convenient user experience. Due to the huge volume of the website (More than 50 pages), we have divided the projects into three stages to address the solutions-

Stage 1

Information Design

Navigation: Add concise mega menu to the website for easy page navigation and access to the information
Content: Structure content based on priority, relevance and purpose of use, and consistency in page layout to make it easy accessible and understandable.

Stage 2

UX Design

Dynamic Interaction: Add multiple dynamic features where necessary to increase interactive contents. Such as: Add dynamic search options and preview to the document rich pages in order to allow users search their desired file.
User Friendly: Create clean concise and interactive design for user friendly interaction.
Responsive Web Design: Make website responsive across all screen sizes.
Usability Testing: Continuously conduct usability testing step by step to create an environment for effortless user interaction.

Stage 3

Development

Develop full functional redesigned website.

SOLUTION

I have followed a set of Gestalt Principles of Design to guide visual design of the final website prototype.

Similarity

Grouped the submenu and category in megamenu by relavancy.

Proximity

Grouped similiar category contents together without using hard borders but including space in between each block to reflect their category structure.

Closure

Used horizontal scroll and carousel to show that there is more to be found if they swipe right.

Continuation

Navigation bar contains tabs selections with a partial option provides an affordance that there are additional options or interactions.
